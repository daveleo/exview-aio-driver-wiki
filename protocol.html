Packet Structure and Parsing

This page mirrors parsing and checksum behavior from device-certify.ts.

1) UDP-like Signature

Valid packets begin with seven 0x55 bytes:

55 55 55 55 55 55 55 ...

2) Reply Code Decoding (Command Code)

Reply code extraction scans packet bytes starting near index 8 for marker 0xD0:

  1. Find first D0 from index 8 up to index 40.
  2. Read next two bytes: low = bytes[i+1], high = bytes[i+2].
  3. Reply code string is high + low in uppercase.
D0 02 C2  => reply code C202

3) Payload Extraction (Tail Payload Pattern)

Payload is detected near the tail using this structure:

00 <len> 00 <data bytes> <checksum>
  • <len> = number of data bytes
  • <data bytes> must be exactly <len> bytes
  • checksum is final packet byte

Concrete examples:

  • Query with 1-byte data: 00 01 00 64 DA (data 64)
  • ACK with 2-byte status: 00 02 00 01 00 7A (status bytes 01 00)

Status decode is little-endian:

status = data[0] | (data[1] << 8)
Status Hex Meaning

4) Checksum (computePdfChecksum)

Normal rule used by most commands:

checksum = (sum of bytes[index 8 .. len-2]) & 0xFF

That means include the 9th byte through the byte before checksum, then mask with 0xFF.

5) Special Formula-base Checksums

Some numeric set commands use:

checksum = (base + value) & 0xFF

    For other numeric commands, use the normal sum rule.
